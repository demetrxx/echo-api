> **Пользователь не должен знать, как у тебя устроен pipeline.**  
> Он должен понимать, *что он контролирует* и *что он получает*.

Поэтому:
- технические сущности → остаются
- продуктовые сущности → переименовываются
- часть вообще исчезает из UI

- если он правит тексты вручную

Микро-copy:

Это **приглашение**, не требование.

## 5. Как Angle влияет на качество (под капотом)

- Angle:
  - перевешивает сигналы
  - влияет на tone
  - снижает “усреднение”
- Если Angle пуст:
  - Echo опирается на:
    - Signals
    - Echo Profile

👉 Качество **растёт со временем**, а не падает из-за пустых полей.

## 3. Типы триггеров обучения (4 категории)

Вот **модель**, которую ты можешь реализовать итеративно.

---

### 🔹 1. Absence-based (отсутствие действия)

**Пример:**
- 0 Angles
- 0 Context views
- 0 Edits
- 0 Feedback clicks

#### UI-паттерн:
- muted inline hint
- не модал
- не tooltip на весь экран

**Пример copy:**
> “You can shape how Echo approaches this theme by adding an angle.”

---

### 🔹 2. Frustration-based (сигнал недовольства)

**Триггеры:**
- 2–3 регенерации подряд
- ручные правки
- быстрый “generate → regenerate”

#### UI-паттерн:
- contextual suggestion

**Copy:**
> “Looks like you’re refining this a lot.  
> Setting an angle could help Echo get it right faster.”

---

### 🔹 3. Curiosity-based (интерес)

**Триггеры:**
- hover
- scroll до конца
- pause

#### UI-паттерн:
- collapsible hint
- “Learn more”

**Copy:**
> “Want to see what shaped this theme?”

---

### 🔹 4. Mastery-based (продвинутый пользователь)

**Триггеры:**
- частое использование
- стабильный usage

#### UI-паттерн:
- subtle badge
- “Advanced”

**Copy:**
> “Power tip: Angles help Echo develop a consistent perspective over time.”

## 5. Важный принцип копирайта

❌ Плохо:
> “You should add an angle”

✔️ Хорошо:
> “You can refine how Echo approaches this theme”

Всегда:
- option
- benefit
- без давления

---

## 6. Как не превратить подсказки в мусор

### Жёсткие правила:
1. Не больше **1 активной подсказки на экран**
2. Одна подсказка = **одна мысль**
3. Подсказка исчезает после действия
4. Можно закрыть навсегда

  - `angle_used`
  - `context_viewed`
  - `regen_count`
- условий показа
- простого inline компонента

Это реально сделать за **1–2 дня**, но эффект огромный.

## 8. Почему это лучше любого онбординга

Пользователь чувствует:
- его не учат
- его не торопят
- продукт “подсказывает вовремя”

👉 **Это ощущается как интеллект продукта**, а не как подсказки.

## 2. Генерация контента на разных языках — MUST HAVE

Это **критично** уже сейчас.

Почему:
- контент-креаторы ≠ только англоязычные
- Telegram / Instagram / X — сильно не-англоязычные
- психологи, эксперты, инфлюенсеры пишут на родном языке
- конкуренты часто хуже работают не на EN

👉 Если Echo не умеет уверенно писать на языке пользователя — продукт мёртв для него.

**Вывод:**  
✔ Content language — **обязательно с v1**

---

## 3. Multilanguage UI — не must have, но сильный буст

Теперь про интерфейс.

### Когда UI НЕ нужен на других языках:
- early adopters
- devs
- founders
- AI-savvy users

Они нормально живут с EN UI.

---

### Когда UI ДАЁТ рост:
- non-tech creators
- психологи / коучи
- инфлюенсеры
- Telegram-first аудитория
- LATAM / Eastern Europe / Asia

И вот тут начинается интересное.

---

## 4. Почему multilanguage UI может быть твоим преимуществом

Большинство AI SaaS:
- ленивы
- делают EN-only
- откладывают i18n “на потом”

Если ты:
- сразу заложишь i18n архитектуру
- начнёшь с 2–3 языков

Ты:
- снижаешь friction
- увеличиваешь активацию
- выигрываешь в retention
